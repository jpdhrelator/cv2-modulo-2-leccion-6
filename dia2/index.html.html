⚽ Mi Dashboard del Mundial 2026

Aplica jQuery, eventos y manipulación del DOM construyendo tu propio panel mundialero.

🎯 Objetivo de la actividad

Construir una mini-aplicación web con temática del Mundial 2026 donde un fanático pueda crear su perfil, calcular puntos en la fase de grupos, predecir el resultado de un partido y cambiar el tema visual de la página.

Toda la lógica debe estar hecha con jQuery, aplicando los eventos vistos en clase (click, change, keyup, hover, focus, blur) y la manipulación del DOM con .val(), .text(), .html(), .css() y .toggleClass().

🗂️ Estructura del proyecto

mundial-2026/
├── index.html       ← estructura HTML
├── estilos.css      ← estilos del dashboard
└── app.js           ← toda la lógica con jQuery

Recuerda incluir jQuery desde el CDN antes de tu app.js:

<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
<script src="app.js"></script>

🧩 Lo que debe tener tu dashboard

Tu página debe tener un título principal y cuatro secciones, cada una con su propia funcionalidad. Las describimos a continuación.

1️⃣ Perfil del hincha keyup · change · click

Una mini ficha donde el usuario ingresa su nombre y elige su selección favorita.

Elementos HTML:
  • Un <input type="text"> para el nombre del hincha.
  • Un <select> con al menos 6 selecciones (por ejemplo: 🇦🇷 Argentina, 🇧🇷 Brasil, 🇨🇱 Chile, 🇺🇾 Uruguay, 🇫🇷 Francia, 🇪🇸 España).
  • Un botón "Crear mi perfil".
  • Un <div> donde se mostrará el resultado.
Comportamiento esperado:
  • Mientras el usuario escribe su nombre, debe aparecer un mensaje en vivo arriba del input que diga: "Hola, [nombre]..." (usa el evento adecuado para detectar la escritura).
  • Cuando cambie la selección en el <select>, debe aparecer (o cambiar) el texto: "Tu equipo: [equipo elegido]".
  • Al hacer click en "Crear mi perfil", se debe armar una tarjeta dentro del div de resultado con el nombre, el equipo elegido y el mensaje "¡Vamos [equipo]! 🏆".
  • Si el nombre está vacío o no se eligió equipo, mostrar un mensaje de error en rojo.

2️⃣ Calculadora de puntos en la fase de grupos click · focus · blur

Una pequeña calculadora que recibe cuántos partidos ganó, empató y perdió un equipo en la fase de grupos, y devuelve los puntos totales.

Recuerda: Victoria = 3 puntos · Empate = 1 punto · Derrota = 0 puntos.

Elementos HTML:
  • Tres <input type="text">: partidos ganados (PG), empatados (PE) y perdidos (PP).
  • Un botón "Calcular puntos".
  • Un <span> o <div> donde se mostrará el total.
Comportamiento esperado:
  • Al hacer click en "Calcular puntos", aplicar la fórmula (PG × 3) + (PE × 1) y mostrar el resultado: "Tu equipo tiene X puntos".
  • Cuando un input recibe el foco, debe cambiar su borde a azul.
  • Cuando un input pierde el foco, debe volver al borde original.
  • Si la suma de PG + PE + PP es mayor a 3, mostrar un mensaje: "⚠️ En la fase de grupos solo se juegan 3 partidos".
  • Si el equipo tiene 7 o más puntos, agregar el texto "¡Clasificado a octavos! 🎉" y pintar el resultado en verde.

3️⃣ Predictor de partido change · click · hover

El usuario elige dos selecciones y predice el marcador. La app le dice quién gana.

Elementos HTML:
  • Dos <select> (equipo local y equipo visitante) con los mismos 6 equipos del punto 1.
  • Dos <input type="text"> para los goles de cada equipo.
  • Un botón "Predecir resultado".
  • Un <div> donde se mostrará la predicción.
Comportamiento esperado:
  • Cuando el usuario pase el mouse sobre el botón, debe cambiar el color del botón (ej: verde más oscuro). Al sacar el mouse, debe volver al color original.
  • Al cambiar cualquiera de los dos <select>, mostrar en un <p> arriba: "Partido: [Local] vs [Visitante]".
  • Al hacer click en "Predecir resultado", mostrar dentro del div:
    • El marcador final: "[Local] 2 - 1 [Visitante]".
    • Quién gana, o si fue empate.
  • Si los dos equipos elegidos son iguales, mostrar el mensaje: "❌ Un equipo no puede jugar contra sí mismo".
Importante: el value del <option> debe ser el nombre del equipo (no un número). Así puedes leerlo con .val() y mostrarlo directamente.

4️⃣ Cambio de tema "Estadio" click · toggleClass

Un botón que cambia toda la página entre dos modos visuales: "Estadio diurno" y "Estadio nocturno".

Elementos HTML:
  • Un botón con el texto inicial "🌙 Modo nocturno".
Comportamiento esperado:
  • Al hacer click, se debe agregar (o quitar) la clase nocturno al <body>.
  • En el CSS, define la clase .nocturno con un fondo oscuro y texto claro.
  • El texto del botón debe cambiar entre "🌙 Modo nocturno" y "☀️ Modo diurno" según el estado.

✅ Checklist antes de entregar

  • El archivo index.html tiene las 4 secciones bien identificadas.
  • jQuery está cargado desde el CDN antes de tu archivo app.js.
  • Todo el código JavaScript usa jQuery (no se usó document.getElementById).
  • La sección 1 reacciona mientras escribes (sin necesidad de presionar el botón).
  • La calculadora de puntos aplica la fórmula correcta y valida los 3 partidos.
  • El predictor muestra el ganador o el empate.
  • El cambio de tema funciona y el texto del botón se actualiza.
  • La página tiene estilos propios (no se ve "pelada").
  • El código está comentado explicando qué hace cada parte.

🌟 Desafíos extra (opcionales)

  1. Que cuando el hincha presione Enter en el input del nombre, se cree el perfil automáticamente (sin tocar el botón).
  2. Agregar una animación con .fadeIn() al mostrar la tarjeta del perfil.
  3. En el predictor, mostrar un emoji 🏆 al lado del ganador.
  4. Agregar un contador que muestre cuántas predicciones lleva hechas el usuario.
  5. Que al elegir un equipo en el perfil, el color de fondo de la tarjeta cambie al color del equipo (ej: 🇦🇷 → celeste).
