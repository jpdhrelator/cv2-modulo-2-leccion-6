Actividad: Conociendo la librería jQuery

Comparando JavaScript puro y jQuery en una mini calculadora

1. Objetivo

Construir una calculadora de precio total que permita al usuario ingresar la cantidad de productos en un input text, elegir un producto desde un select (cada opción con su precio) y, al presionar un botón, ejecutar una función que multiplique la cantidad por el precio y muestre el total directamente en el HTML.

Debes resolver el ejercicio dos veces: una usando solamente JavaScript puro y otra usando jQuery. Ambas versiones deben entregar el mismo resultado.

2. Descripción del ejercicio

Construye una calculadora de precio total con los siguientes elementos:

  • Un <input type="text"> para ingresar la cantidad de productos.
  • Un <select> con al menos 4 productos, cada uno con su precio en el atributo value.
  • Un <button> con el texto "Calcular total".
  • Un elemento (por ejemplo un <span> o <div>) donde se mostrará el resultado.

Al hacer click en el botón, una función debe leer la cantidad y el precio, multiplicarlos y mostrar el total dentro del elemento de resultado.

Importante: el HTML debe contener dos secciones (una para JS puro y otra para jQuery), cada una con sus propios inputs, select, botón y resultado. Usa id distintos en cada sección.

3. Estructura de archivos sugerida

ejercicio-jquery/
├── index.html
├── estilos.css         (opcional)
├── js-puro.js          ← tu código en JavaScript vanilla
└── jquery-version.js   ← tu código usando jQuery

4. Requisitos técnicos

Parte A — JavaScript puro

  • Selecciona los elementos del DOM usando document.getElementById().
  • Lee el valor del input y del select con la propiedad .value.
  • Asocia el evento click al botón usando el atributo onclick del HTML (por ejemplo <button onclick="miFuncion()">) o la propiedad boton.onclick = miFuncion; en el JS.
  • Modifica el HTML con .textContent o .innerHTML.
  • No uses jQuery en esta parte.

Parte B — jQuery

  • Incluye jQuery con el CDN: <script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
  • Selecciona los elementos con $("#id").
  • Lee los valores con .val().
  • Asocia el evento click con .click(function(){ ... }).
  • Modifica el HTML con .text() o .html().
  • Envuelve tu código en $(document).ready(function(){ ... });
Resultado esperado: ambas secciones deben mostrar el mismo total al ingresar los mismos datos.

5. Pista: equivalencias entre JS puro y jQuery

Acción JavaScript puro jQuery
Seleccionar elemento document.getElementById("id") $("#id")
Leer valor el.value $("#id").val()
Cambiar texto el.textContent = "..." $("#id").text("...")
Evento click boton.onclick = fn $("#id").click(fn)

6. Validaciones que debes considerar

  • Si la cantidad está vacía o no es un número, debe mostrarse el mensaje "Datos inválidos".
  • Si no se ha seleccionado un producto del select, también debe mostrar "Datos inválidos".
  • El total debe mostrarse con el signo $ al inicio.

7. Desafíos extra (opcional)

  1. Aplica un descuento del 10% si la cantidad es mayor a 5 unidades.
  2. Si el total supera los $10.000, pinta el resultado de color rojo (usa .style.color y .css("color", ...) según corresponda).
  3. Implementa el evento change en el select para que el total se calcule automáticamente al cambiar de producto, sin necesidad de presionar el botón.

8. Checklist de entrega

  • El archivo index.html contiene las dos secciones (JS puro y jQuery).
  • El archivo js-puro.js resuelve la actividad sin usar jQuery.
  • El archivo jquery-version.js resuelve la actividad usando jQuery.
  • jQuery se carga desde el CDN antes de los scripts propios.
  • Ambas calculadoras entregan el mismo resultado con los mismos datos.
  • El código está comentado explicando qué hace cada paso.
Recuerda el lema de jQuery: "Write less, do more". Al terminar, compara cuántas líneas usaste en cada versión.
